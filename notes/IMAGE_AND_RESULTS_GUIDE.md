## CSV Files (Always Save)

These are generated by notebook cells and should **always** be exported to `New_Changes/results/`:

```python
# In notebooks, use:

# From yield_changes/ subdirectory:
df.to_csv('../results/Filename.csv', index=False)
```

## PNG/Image Generation & Saving

### Option A: Save directly in notebook

```python
import matplotlib.pyplot as plt

plt.savefig('../results/MyPlot.png', dpi=300, bbox_inches='tight')
plt.show()
```

### Option B: Plotly exports

```python
import plotly.graph_objects as go

fig = go.Figure(...)
fig.write_html('../results/MyPlot.html')        # Interactive
fig.write_image('../results/MyPlot.png')        # Static (requires kaleido)
```

## Loading Images in Streamlit

From the `pages/*.py` files, images are loaded like:

```python
from pathlib import Path

def resolve_results_png(filename):
    """Resolve image path from results directory."""
    return Path(__file__).parent.parent / "New_Changes" / "results" / filename

# Usage:
img_path = resolve_results_png("Crop_Climate_Sensitivity.png")
if img_path.exists():
    st.image(str(img_path), caption="Crop Sensitivity to Climate", use_container_width=True)
```

**Key point:** Streamlit pages expect images in `New_Changes/results/` relative to the app root.

## Troubleshooting

### "Image not found" in Streamlit

**Check:**
1. Is the file actually in `New_Changes/results/`?
2. Is the filename exactly correct (case-sensitive)?
3. Run: `ls New_Changes/results/` in terminal to list files

### CSV loading errors

**Check:**
1. File is saved with `index=False` to avoid unnamed index column
2. File exists at the path specified in `data_loader.py`
3. No special characters in column names; use underscores

### PNG quality issues

**Recommendations:**
- Use `dpi=300` for publication-quality images
- Use `bbox_inches='tight'` to remove whitespace
- For large plots, use `figsize=(12, 8)` or larger
